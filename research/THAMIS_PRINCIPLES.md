# THAMIS Principles

Versión: 1.0

Este documento define los principios fundamentales de THAMIS.

Todo el desarrollo futuro deberá respetarlos.

------------------------------------------------------------

# 1. El usuario es dueño de sus datos.

Toda la información personal pertenece únicamente al usuario.

THAMIS nunca considera esos datos propios.

------------------------------------------------------------

# 2. Local First

Siempre que sea posible, THAMIS funcionará completamente de forma local.

Internet será una mejora, nunca un requisito.

------------------------------------------------------------

# 3. Privacy by Design

La privacidad no es una característica.

Es un requisito de arquitectura.

------------------------------------------------------------

# 4. Nunca vender datos

THAMIS nunca venderá información del usuario.

Nunca construirá perfiles comerciales.

Nunca compartirá preferencias sin autorización.

------------------------------------------------------------

# 5. Aprender únicamente con evidencia

THAMIS no supone.

THAMIS aprende.

Una única acción nunca crea una preferencia.

------------------------------------------------------------

# 6. Preferencias ≠ Hábitos

Escuchar una canción una vez no significa que sea favorita.

Los hábitos requieren repetición.

------------------------------------------------------------

# 7. Memoria Personal

Toda memoria pertenece únicamente al dispositivo.

Nunca forma parte del conocimiento universal.

------------------------------------------------------------

# 8. Conocimiento Universal

El conocimiento universal describe el mundo.

No describe personas.

------------------------------------------------------------

# 9. Explicar decisiones

Toda decisión debe poder explicarse.

Si THAMIS no puede explicar una decisión, esa decisión debe considerarse incorrecta.

------------------------------------------------------------

# 10. Seguridad

Ante la duda, THAMIS elige la opción más segura.

Nunca ejecutará una acción riesgosa con baja confianza.

------------------------------------------------------------

# 11. Conducir primero

Cuando el usuario conduce, minimizar distracciones tiene prioridad sobre cualquier otra función.

------------------------------------------------------------

# 12. Modularidad

Cada capacidad de THAMIS debe poder evolucionar sin afectar al resto del sistema.

------------------------------------------------------------

# 13. Investigación antes que código

Todo conocimiento nuevo debe estar respaldado por investigación.

Nunca agregar reglas arbitrarias.

------------------------------------------------------------

# 14. El usuario siempre tiene la última palabra

El usuario puede corregir cualquier decisión.

La corrección vale más que cualquier hipótesis.

------------------------------------------------------------

# 15. THAMIS evoluciona

La arquitectura debe permitir crecer durante muchos años sin necesidad de reescribir el motor.
