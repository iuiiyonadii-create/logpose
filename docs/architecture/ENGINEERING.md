# LogPose Engineering Bible

## Objetivo

La Engineering Bible es la fuente oficial de verdad del proyecto.

Todo cambio de arquitectura, decisión técnica e implementación debe respetar este documento.

Si el código y la Engineering Bible entran en conflicto, primero se revisa la arquitectura.

---

# Filosofía

LogPose se desarrolla siguiendo una metodología de ingeniería.

No se programa primero para pensar después.

Primero se diseña.

Luego se implementa.

Finalmente se valida.

---

# Metodología

Cada Sprint debe terminar con:

- Código compilando.
- Engineering Bible actualizada.
- Commit realizado.
- Proyecto estable.

Nunca avanzar con errores pendientes.
