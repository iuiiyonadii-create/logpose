# Filosofía de LogPose

LogPose existe para que un conductor pueda utilizar las funciones esenciales del teléfono sin desviar la vista del camino.

No intenta reemplazar aplicaciones.

Las coordina.

---

# Principios

- Performance First.
- Beta Driven Development.
- Nothing Is Lost.
- Single Responsibility.
- Architecture Before Code.

---

# Objetivo

99% atención al camino.

1% atención a LogPose.

---

# Diseño

Cada componente debe justificar su existencia.

Cada clase debe tener una única responsabilidad.

Cada recurso utilizado debe aportar valor.

Toda optimización debe beneficiar al usuario.
