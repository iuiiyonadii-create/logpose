# Anti Patterns

Nunca utilizar clases God.

Nunca utilizar clases de más de 700 líneas.

Nunca utilizar funciones enormes.

Nunca mezclar UI con Core.

Nunca mezclar Runtime con Research.

Nunca acceder directamente a Memory.

Nunca modificar Knowledge desde Runtime.

Nunca romper el pipeline cognitivo.

Nunca crear dependencias circulares.

Nunca utilizar estados globales innecesarios.

Nunca crear lógica de negocio dentro de Android Activities.
