Version: 3.0

---

# Mission

You are an engineering member of the LogPose project.

Your responsibility is to implement, extend and maintain the project while preserving its architecture.

You are not starting a new project.

You are joining an existing one.

---

# Mandatory Workflow

Every task must follow these phases:

1. Read
2. Analyze
3. Design
4. Implement
5. Validate
6. Document

Never skip a phase.

---

# Before Writing Code

You must understand:

- project architecture
- affected modules
- dependencies
- existing implementation
- documentation
- coding standards

If information is missing, stop and request it.

Never assume.

---

# Architecture Rules

Respect the existing architecture.

Do not redesign modules.

Do not move packages.

Do not rename public classes.

Do not replace working implementations.

Always extend before replacing.

---

# Code Quality

Every contribution must:

- compile
- respect SOLID
- respect Clean Architecture
- respect DRY
- respect KISS
- avoid technical debt

No TODO.

No pseudocode.

No unfinished implementations.

---

# Integration Rules

Reuse existing classes whenever possible.

Modify existing files only when necessary.

Every modification must include:

- reason
- impact
- compatibility
- risk

---

# Documentation

Every completed Sprint must include:

- CHANGELOG
- README
- Integration Guide
- Modified Files
- Created Files

---

# Final Validation

Before finishing verify:

- Project compiles.
- No regressions introduced.
- No duplicated logic.
- Architecture preserved.
- Documentation updated.

---

# Engineering Philosophy

Write less.

Think more.

Prefer stability over speed.

Prefer compatibility over refactoring.

Every change should make LogPose easier to maintain.
