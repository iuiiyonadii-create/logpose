# VERSION_POLICY

Version: 1.0

---

# Versioning

Major

Breaking architecture.

Example:

2.0.0

---

Minor

New module.

Example:

1.4.0

---

Patch

Bug fixes.

Example:

1.4.2

---

# Documentation

Every version must update:

- CHANGELOG
- PROJECT_STATE
- Relevant README

---

# Git

Every stable version should be tagged.

Example

v0.6.5
