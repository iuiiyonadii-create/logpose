# CODING_STANDARD

Version: 1.0

Status: OFFICIAL

------------------------------------------------------------------------------

# PURPOSE

Defines the coding standards for the LogPose project.

------------------------------------------------------------------------------

# LANGUAGE

Primary language:

Kotlin

UI:

Jetpack Compose

------------------------------------------------------------------------------

# PRINCIPLES

Write readable code.

Write maintainable code.

Write testable code.

Prefer simplicity.

------------------------------------------------------------------------------

# CLASS DESIGN

One class.

One responsibility.

One purpose.

------------------------------------------------------------------------------

# FUNCTIONS

Functions should:

- have one responsibility
- be small
- have descriptive names
- avoid side effects

------------------------------------------------------------------------------

# NAMING

Classes:

PascalCase

Functions:

camelCase

Variables:

camelCase

Constants:

UPPER_SNAKE_CASE

Packages:

lowercase

------------------------------------------------------------------------------

# COMMENTS

Comment WHY.

Do not comment WHAT.

Good code explains itself.

------------------------------------------------------------------------------

# FORBIDDEN

Magic numbers.

Dead code.

TODO in production.

Unused imports.

Duplicated logic.

------------------------------------------------------------------------------

# REQUIRED

Meaningful names.

Small classes.

Low coupling.

High cohesion.

SOLID.

DRY.

KISS.

------------------------------------------------------------------------------

END OF DOCUMENT
