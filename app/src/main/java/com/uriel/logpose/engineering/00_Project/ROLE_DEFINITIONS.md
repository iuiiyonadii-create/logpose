# ROLE_DEFINITIONS

Version: 1.0
Status: OFFICIAL

---

# Purpose

This document defines the responsibilities of every participant in the LogPose project.

---

# Project Owner

Responsibilities:

- Defines product vision.
- Sets priorities.
- Approves releases.
- Accepts Sprint completion.

Cannot:

- Change architecture directly.

---

# CTO / Architect

Responsibilities:

- Owns project architecture.
- Defines engineering standards.
- Approves architectural changes.
- Creates Sprint specifications.
- Reviews integrations.

Has final authority over technical decisions.

---

# Builder

Responsibilities:

- Implements approved Sprint work.
- Writes production-ready code.
- Maintains compatibility.
- Adds tests.
- Updates implementation.

Cannot:

- Redesign architecture.
- Rename packages.
- Replace modules without approval.

---

# Reviewer

Responsibilities:

- Reviews implementation.
- Detects regressions.
- Finds technical debt.
- Verifies standards.
- Suggests improvements.

---

# Documentation

Responsibilities:

- Maintains documentation.
- Updates CHANGELOG.
- Writes integration guides.
- Synchronizes documentation with implementation.

---

# Rule

Every role must respect PROJECT_GOVERNANCE.md.
